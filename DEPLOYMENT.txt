LW-app production checklist
===========================

1) Supabase — two connection strings (Project → Connect → Connection string)

   DATABASE_URL — use the TRANSACTION pooler on port 6543 (not 5432):
   postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres

   DIRECT_URL — direct connection (migrations / Prisma):
   postgresql://postgres:[PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres

   The API code appends sslmode=require and (for :6543) pgbouncer=true + connection_limit=1 automatically.
   If login still fails, switch DATABASE_URL to port 6543 — Prisma + Supabase session pooler :5432 is unreliable from some hosts.

2) One-time schema (local machine, server/.env filled)
   cd /path/to/lw-app && npm run db:push

3) Railway (API)
   - Deploy from GitHub; Dockerfile.api at repo root; railway.toml points to it.
   - Variables: DATABASE_URL, DIRECT_URL, JWT_SECRET, NODE_ENV=production,
     WEB_ORIGIN=https://livingwordpnw.vercel.app
   - After deploy, verify:
     https://YOUR-RAILWAY-URL/health     → {"ok":true}
     https://YOUR-RAILWAY-URL/health/db → {"ok":true,"db":true}
   If /health/db returns 503, DATABASE_URL/DIRECT_URL are wrong — fix in Railway Variables and redeploy.

4) Vercel (web)
   - API_PROXY_TARGET=https://YOUR-RAILWAY-URL (no trailing path)
   - Redeploy production.

Troubleshooting
---------------
• Vercel build failed with Prisma/tsc errors in `server/`: the repo root must NOT run `tsc` on Vercel — only the Next app (`web/`) should build. Root `vercel.json` fixes this.
• Railway `/health` = 200 but `/health/db` shows “Circuit breaker open” or “upstream database”: the Supabase *pooler* cannot reach Postgres. In Supabase Dashboard, confirm the project is **not paused** (free tier). Then set Railway `DATABASE_URL` to the **Direct connection** string (host `db.<ref>.supabase.co:5432`), not the pooler — one Railway process can use a direct connection safely.
• Vercel “Authentication Required” on some URLs: turn off **Deployment Protection** for Production (Project → Settings → Deployment Protection), or test on the primary production domain only.

Notes
- Do not commit server/.env.
- Supabase “API Keys” tab is not used by this Express+Prisma API.
